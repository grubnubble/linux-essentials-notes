Role of an Operating System (OS)

note: an operating system is not an application

OSs provide 5 key functions in a computer system:

  1. provides an application platform; the OS manages applications' access to central processing unit (CPU) and system memory
  2. serves as a hardware moderator to manage how applications use the computer's hardware

    eg: OS ensures that one application doesn't try to use memory that is already being used by another application

  3. Uses a filesystem to provide efficient and reliable means for data storage
  4. provides security for its data, to varying degrees depending on the OS
  5. manages connectivity to other devices

The Linux Operating System is composed of:

  • Linux kernel: the actual OS itself, the heart of Linux
  • Libraries: prewritten code elements that are shared with all applications
  • Utilities: for completing OS management tasks

    egs: maintaining filesystem, editing text files, managing running processes, installing new packages

  • User Interface: command line interface (CLI) and optional graphical user interface (GUI

How Linux Came to Be

a grad student in Helsinki, Finland, named Linus Torvalds developed the Linux kernel based on Minix

Minix was a clone of Unix; it was developed by Dr. Andrew S. Tanenbaum so that he could teach his students about the inner workings ofo an OS; he wanted them to be able to see the source code; it was a minimal freeware OS

At the time, there were only three main OSs: DOS, Mac OS X, and Unix

  • all three were commercially developed, protected by patents and copyrights
  • it was not possible to reverse-compile these OSs (reverse-compiling is a process of generating source code, given a binary program file. There are applications that can do this.)

in 1991, Torvalds cloned Unix and created Linux. It consisted only of the Linux kernel and three utilities: Bourne again shell bash (commandd line interface), update (for flushing file buffers, whatever that means), and gcc (a C++ compiler)

Torvalds posted the source code and invited programmers to improve it

GNU and Linux

The GNU's Not Linux (GNU) movement:

  • in the 1980s, Rishard Stallman of MIT objected the to proprietary nature of the corporate software development model
  • the idea is that the source code should be freely distributable in order to encourage developers to make improvements to it
  • gcc was developed under GNU

Linux is licensed under GNU General Public License (GPL) so that anyone can use, modify, or distribute its source code

Distributions

take your basic Linux OS recipe (ie the source code for the kernel) and add some things (namely tools, utilities, applications) to get different flavors of Linux

Linux is unique in the OS world for having so many distributions developed by many different people and companies

there are hundreds of Linux distributions; most are free but some must be paid for

Life Cycle of a Linux Distribution

essentially the same life cycle as any other software:

  1. Design: the goal or purpose of pthe distribution is identified
  2. Develop: the distribution is created according to the plans made in the design phase; usually debugged
  3. Deploy: the completed distribution is released
  4. Manage: bugs are identified; updates are made to fix them every few months
  5. Retire: after support is pulled (ie: after the end of the suppor life cycle), the distribution is obsolete and is retired

It's possible (and probable) that newer versions of a particular distribution will come out before the first version is retired. Cool. The different versions will be at different stages of the support life cycle,, though.

What is an Operating System?

provides all the most fundamental features of a computer, software-wise

enables the use of hardware devices, defines user interface standards, provides basic tools

What is a Kernel?

a software compoment responsible for managing low-level features:

  • interfacing with hardware devices
  • allocating memory to programs
  • allocating CPU time to programs
  • enabling programs to interact with each other

the kernel is the basis; other programs run on top of it

the kernel is the software "glue" that holds the computer together

kernels are not interchangeable: they have different internal designs, provide different software interfaces

the Linux kernel is called Linux (technically Linux only refers to the kernel

What Else Identifies an Operating System?

  • Command line shells: eg Bourne again shell (bash)
  • GUI
  • Utility programs: egs: calendars, calculators, text editors, disk maintenance tools, etc.
  • Libraries: provide critical services to programs; collections of programming functions that can be used by many different programs; eg: libc
  • Productivity programs: web browsers, word processors, etc

Operating systems can be distinguished also by details of user accounts, rules for naming disc files, details of the computer's start up

  • these details are sometimes controlled by the kernel, sometimes not

Comparing Linux to Unix

Linux is modeled after Unix

Unix was created in 1969 at AT&T's Bell Labs

some open source projects that were cloned from or made for Unix include:

  • the linux kernel: designed to be compatible with other Unix kernels
  • the GNU project: an effort by the Free Software Foundation (FSF) to make open source versions of core Unix elements note: GNU HURD is a kernel, but it's not as popular as Linux
  • Xorg-X11: a GUI environment for most Unix OSs
  • desktop environments: such as GNOME, KDE, Xfce - same as Linux OSs
  • server programs: Unix and Linux are both popular as server OSs - for web servers, email servers, file servers, etc.
  • user productivity programs: similar or the same as those in Linux OSs, but more available to Linux

Linux is most similar to Unix than other OSs

Comparing Linux to Windows

this is the most likely comparison

they have similar capabilities, but the details are significantly different:

  • Licensing: Linux is open source; Windows is proprietary
  • Costs: many Linux distributions cost nothing, but it takes effort to maintain and install them
  • Hardware Compatibility: most hardware manufacturers provide drivers for Windows; Linux drivers may take a few weeks, months, or years to appear after the actual hardware does; Linux maintains old drivers for a longer time; Linux is less resource intensive
  • Software Availability: there are many desktop apps that are available for Windows but not Linux; on the server side, the situation is reversed (cf. Apache web server)
  • User Interfaces: Windows has its own interface, making it less portable
  • Configurability: Linux is much more configurable
  • Security: both of these OSs claim to be more secure than the other, depends on how you are looking at security; most viruses are designed for Windows, so they are essentially a non-issue with Linux though

Linux is less popular on desktops (and its apps), but dominates the server market

Understanding Release Cycles

after its release, a distribution is typically supported until some time after the next version's release (usually a few months to a year or more), during which the maintainers provide updates and bug fixes

after the support period ends, you'll have to do those fixes yourself, so it's generally a good idea to upgrade to the newest version

rolling release cycle: upgrades can occur in an ongoing manner, making it unnecessary to ever do a full upgrade; some of the specific upgrades can as a result be more intrusive however

there are two classifications of pre-release versions:

  • Alpha software: extremely new, likely to contain bugs
  • Beta software: more stable than Alpha, but still likely to be buggy

generally, unless you want to contribute to a distribution's development, don't use pre-release versions

For Virtualization

traditionally, one OS is installed on one device, which has full reign over all resources on the server hardware succh as RAM, processor time, storage devices, and network interfaces

however, the OS doesn't fully utilize all of these resources all of the time - this means there are wasted resources; usually each resource is at about 8-9% usage capacity

virtualization pools multiple OS instances onto the same physical hardware and lets them run concurrently, thus making the use of the hardware more efficcient

to do this, virtualization uses a hypervisor to manage access to system resources

virtualization allows multiple platforms to run at the same time on the same hardware

virtualization is a key component of cloud computing

Using Open Source Software

Understanding Basic Open Source Principles (4 basic types)

  1. Commercial software: the intention with commercial software is to make a profit; the source code is generally kept secret; redistribution is generally illegal
  2. Shareware software: similar to commercial software in that it's copyrighted and the author asks for payment, but shareware is sold on an honor system; like Sublime Text
  3. Freeware: available for free, authors don't ask for payment; generally comes without source code
  4. Open source software: there are ten principles that define open source; some main ones: the right of the user to redistribute the program, the right of the user to make and distribute changed versions of the program; source code is available

there are hybrids of each of these types

Arguments for (and Against) Open Source:

  • better code: the code can be peer-reviewed, bugs fixed, improvements made; not all projects are treated equally/give equal attention
  • more flexibility: users are able to customize to their hearts' content and can release these new changes into the community so that others might also benefit
  • lower cost: free!
  • lack of vendor lock-in: can be modified to fit any standards

the downsides are generally regarded as minor compared to the advantages

Understanding OS Roles

three basic roles Linux is useful for:

  1. Embedded Computers: these are specialized devices that fulfill a specfic purpose
    • cell phones - Android
    • e-readers - custom Linux or Android
    • DVRs - Linux
    • car computers - for GPS, radio, internet, engine monitor and control
    • appliances - TVs, fridges
    • embedded devices are designed to be used and not maintained by the end user
  2. Desktop and Laptop Computers: similar to workstations, though workstations tend to be more powerful and specialized
    • open source GUI apps tend to lag behind commercial ones because techies lack incentive on their off time to make apps that work more user-friendly (cf. the monetary incentives for commercial developers)
  3. Server Computers: servers can use the same hardware as desktops, though it is common to have larger hard disks and better network connections
    • most do not require a GUI: so servers do without X, desktop environments, or typical desktop programs
    • those who maintain large servers are usually very technically proficient and can contribute directly to the open source server projects that they use
